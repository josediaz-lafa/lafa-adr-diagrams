ADR-011

Payroll Period Lifecycle

State machine: open through closed with revert paths

stateDiagram-v2
    [*] --> open
    open --> draft : auto-draft calculations

    draft --> reviewed : submit for review
    draft --> open : revert for recalculation

    reviewed --> approved : approve payroll
    reviewed --> draft : corrections needed

    approved --> closed : Runa export confirmed
    closed --> [*]

Decision: 5-state machine with revert paths. Why: Corrections happen every week — the system must allow draft → review → approve without losing data.

ADR-016

Runa Export Lifecycle

State machine for payroll export — pending through confirmed, with discrepancy path

stateDiagram-v2

    pending --> exported : CSV generated and downloaded

    exported --> confirmed : Accountant verifies in Runa
    exported --> discrepancy : Mismatch found

    discrepancy --> pending : Corrections made, re-export

Decision: 4-state export lifecycle with discrepancy detection. Why: The accountant validates in Runa — mismatches must loop back, not disappear.

Decision: 6-state lifecycle from pending to completed/defaulted. Why: Each state triggers different obligations — legal, financial, and operational.

LTO Credit Core

Collections Escalation Workflow

State machine for delinquent contracts — rehabilitative approach, not punitive

stateDiagram-v2
    [*] --> reminder
    reminder --> warning : No response

    warning --> hold : Continued non-payment
    warning --> cured : Payments caught up

    hold --> recovery : Vehicle recovery initiated
    hold --> cured : Payments caught up

    recovery --> recovered : Vehicle recovered
    recovery --> cured : Payments caught up

    recovered --> [*]
    cured --> [*]

Decision: Escalation from reminder to recovery with cure paths at every stage. Why: Rehabilitative, not punitive — we want the driver back in the car, not the car back in the lot.
